O separador **Modelos** lista os textos reutilizáveis que a equipa usa em envios manuais e em alguns fluxos
automáticos. Cada linha mostra o nome, a categoria, os canais suportados e a data da última atualização.

## O que existe nesta lista

- **Modelos de marketing** para comunicações mais abertas ou promocionais.
- **Modelos transacionais** para mensagens ligadas a uma ação concreta do sistema.
- **Modelos bloqueados na estrutura** quando o fluxo depende de um formato controlado.

## Ações disponíveis

<Steps>
<Step>
**Novo modelo** abre o editor para criar um modelo a partir do zero.
</Step>
<Step>
**Usar** abre o assistente de envio já com esse modelo selecionado.
</Step>
<Step>
**Duplicar** cria uma cópia editável, útil quando quer partir de um texto já aprovado.
</Step>
<Step>
**Arquivar** retira o modelo da lista ativa quando ele já não deve ser usado.
</Step>
</Steps>

<Callout title="Modelos do sistema">
Nem todos os modelos podem ser arquivados. Alguns existem para suportar automatismos do próprio produto, por exemplo
os lembretes de regularização, e por isso permanecem protegidos.
</Callout>

## Quando usar um modelo

Use um modelo quando:

- a mesma mensagem vai ser enviada mais do que uma vez;
- o clube quer manter um tom e estrutura consistentes;
- a mensagem precisa de ser revista antes de começar a ser usada em massa.
