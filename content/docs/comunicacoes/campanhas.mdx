title: Campanhas

---

O separador **Campanhas** serve para fluxos estruturados com arranque numa data definida, em vez de um disparo único.
Cada campanha tem nome, categoria, data de início e um estado próprio.

## Estados que pode encontrar

| Estado | Significado |
|---|---|
| **Rascunho** | A campanha ainda está a ser preparada. |
| **Agendada** | Já tem data de início, mas ainda não começou. |
| **Em execução** | A campanha está ativa. |
| **Em pausa** | A campanha foi interrompida temporariamente. |
| **Concluída** | A sequência terminou. |
| **Cancelada** | A campanha foi terminada antes do fim. |

## Como criar uma campanha simples

<Steps>
<Step>
Abra **Comunicações → Campanhas**.
</Step>
<Step>
Clique em **Nova campanha**.
</Step>
<Step>
Defina um nome interno claro e escolha a categoria certa.
</Step>
<Step>
Associe a audiência e o modelo de comunicação a usar em cada passo.
</Step>
<Step>
Escolha uma data de início e reveja a sequência completa.
</Step>
<Step>
Depois de ativar, acompanhe o resultado em [Histórico](/docs/comunicacoes/historico).
</Step>
</Steps>

## Quando usar uma campanha em vez de um envio

Use **campanha** quando:

- quer uma sequência com mais do que um passo;
- precisa de agendar o arranque para um momento futuro;
- quer acompanhar esse conjunto como uma entidade própria.

Use **envio manual** quando:

- precisa de disparar uma comunicação única;
- não há sequência nem calendário para manter.
